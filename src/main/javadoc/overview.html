Getting Started

Step 1: Provide Host and API Token

For the client to access the credentials you need to specify the CommandK Host along with the API Token that was generated in the admin panel. If you instantiate the client with CommandKClient.withDefaults() then the DefaultCredentialProviderChain will be used which looks for the Host and API token in the following order:

  1. Environment variables: It will pick up the values from the environment variable COMMANDK_HOST and COMMANDK_API_TOKEN
  2. System properties: It will pick up the values from the system properties commandk.host and commandk.apiToken
  3. Configurations file: It will pick up the values from a configuration file specified through the environment value COMMANDK_CONFIG_FILE or the system property commandk.configFile

Note: If you provide a configuration file then it must be of the following format:

host=<host>
apiToken=<token>

Step 2: Initialize the client and query for credentials

You can create an instance of the CommandKClient with CommandKClient.withDefaults(), or you can create one by passing your own implmentations of a KVStore and a CommandKCredentialsProvider. 

CommandKClient commandKClient = CommandKClient.withDefaults();
List<RenderedAppSecrets> credentials = commandKClient.getRenderedAppSecrets("appId", "environment");
